# Contributing to UEDGE

## Quick Start
1. Create a feature branch from `develop` (v8) or `v9-develop`
2. Make focused commits
3. Rebase before pushing
4. Open a PR into your feature branch

## Rules
- Never commit to main or develop
- Feature branch assumed to be v8 contributions unless names prefaced with v9-
- Follow backport labeling rules

Please consult [doc/workflow.md](doc/workflow.md) for detailed contribution workflow.
